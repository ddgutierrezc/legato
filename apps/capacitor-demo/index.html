    Remote transport v2 validation checklist

    • Background play/pause parity: with playback running, background app and toggle lock-screen/notification play↔pause; verify state changes match manual play()/pause().
    • Remote next/previous parity: on lock-screen/notification and manual buttons, next/previous must match queue transitions from skipToNext()/skipToPrevious().
    • Remote seek parity: lock-screen/notification seek must match seekTo() and keep snapshot position aligned.
    • Boundary behavior parity: previous at first track restarts to 0, next at last track transitions to ended + playbackEnded event.
    • Capability projection parity: canSkipNext/canSkipPrevious/canSeek summary must align with queue index + ended/empty states.
    • Android lockscreen/notification position from real snapshot: on resume/seek, projected position should rebase near snapshot target (no transient 0 reset).
    • iOS now-playing playbackRate fidelity: expect playbackRate=1.0 while playing and playbackRate=0.0 when paused/stopped/non-playing.
    • Now-playing metadata parity: verify title/artist/album/artwork/duration are visible in lock screen / media notification.
    • Artwork replacement/stale guard: run Run artwork race, then verify lock-screen/notification artwork matches the final active track (no stale previous image).
    • Artwork fallback clear: during the race flow, track 3 has no artwork; verify lock-screen/notification clears artwork before switching back to an artwork track.

    After harness updates: run npm run build and npm run cap:sync before Android Studio/Xcode validation.
