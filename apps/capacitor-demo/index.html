Remote parity validation checklist

  • Background play/pause parity: with playback running, background app and toggle lock-screen/notification play↔pause; verify state changes match manual play()/pause().
  • Now-playing metadata parity: verify title/artist/album/artwork/duration are visible in lock screen / media notification.
  • Progress parity: while playing, lock-screen/notification progress advances; after pause it stays frozen at pause point.
  • v1 command scope: next/previous/seek from remote surface must be inert (no crashes, no track/position jump).

After harness updates: run npm run build and npm run cap:sync before Android Studio/Xcode validation.
